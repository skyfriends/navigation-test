3  APM Screens

3.11 Alarm Summary Screen

3.11.1 Section Overview

The Alarm Summary screen displays all currently active alarms, both unacknowledged and acknowledged. The Alarm Summary screen is accessed by clicking the “(F9) Alarm Summary” button in the region of the Main screen with the gray background color. It can also be accessed from any screen by pressing the “F9” function key on the APM console keyboard.

3.11.2 Details

An example of an alarm summary is reproduced in Figure 3.100. All currently active alarms are displayed here. Unacknowledged alarms are displayed in red text; acknowledged alarms are displayed in yellow text. The most recent alarms are displayed at the top of the list; the oldest alarms are displayed at the bottom of the list. If there are more active alarms than there are rows on the list, the operator can use the vertical scroll bar to scroll down to view the older alarms.

The aspects of the Alarm Summary screen that will be important for the operator to know are as follows:

  1. “Refresh” Button - When the Alarm Summary screen is selected from the Main screen it displays all alarm log entries up to the moment that the screen appears. The alarm list does not refresh itself as new alarm events occur. To do that the operator must click the Refresh button.
  2. Event Time Column - The date and time when the alarm was originally raised (for unacknowledged alarms) or acknowledged (for acknowledged alarms).
  3. Alarm Name Column - Displays the reference number for each alarm.
  4. Message Column - Displays the description for each alarm.
  5. Total Number of Unacknowledge Alarms - Displays the total number of currently unacknowledged active alarms.
  6. Total Number of Acknowledge Alarms - Displays the total number of currently acknowledged active alarms.
  7. “Acknowledge All Alarms End” Button - Clicking this button acknowledges all currently unacknowledged active alarms. (The operator can also acknowledge by pressing the End key on the APM console keyboard from any screen.)
  8. “Troubleshooting” Button - Clicking this button brings up the Troubleshoot screen.
  9. “Close” Button - Clicking this button closes the Alarm Summary screen.

Be sure to scroll through and note all alarms in the alarm list before acknowledging alarms. The reason for this is that acknowledging alarms first will result in any alarm for which the condition that originally caused that alarm to be raised has been resolved, the alarm message will disappear before the operator will have a chance to see it.
