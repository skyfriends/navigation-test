3 APM Screens

3.1 Information Common to All Screens

3.1.1 Section Overview

This section addresses some of the Automated Process Management (APM) software features that are common to many of the major APM screens. First discussed are the APM header and status bar, two features that are visible regardless of which APM screen is selected for display. Next discussed are the shapes that are commonly used to represent various types of Wenger extruder system components, such as motors, valves, pumps, etc. Also discussed are the graphical color conventions that are used to quickly convey status information of the various extruder system components. Finally, there is an extensive and detailed explanation of the various types of Hand-Off-Auto (HOA) faceplate’s that operators will frequently use to configure automatic control of the extruder system by the PLC. These classes of HOA faceplate’s are for fixed speed motors, variable speed motors, on/off valves, proportional flow control valves, and temperature controllers.

3.1.2 Header and Status Bar

There are two regions that are always visible on the APM monitor, regardless of which APM screen is being displayed. These are the header and the status bar (see Figure 3.1).

  1. Header Bar - This thin rectangular region that runs horizontally along the top of the monitor from the left edge to the right edge is itself composed of three fields that will be described in more detail below.
  2. APM Status Bar - This rectangular region along the bottom of the monitor displays various diagnostic messages.

3.1.2.1 Main Screen Header - Left Zone

The left side of the header displays the system status of the extruder (see Figure 3.2). System status (1) and (2) is an indication of the current overall operating state of the extruder system. For example, when the extruder has completed its startup sequence and is running, this region of the Header will display “Extruder Running”. When the system is not running, or has completed its shutdown sequence, this region of the Header will display “Extruder Stopped”. The beginning of specific key events in the startup and shutdown sequence will be displayed during the startup and shutdown of the system. Below the system status area is where information is displayed pertaining to the formula presently used to control the extruder. Both formula name (3) and revision (4) are displayed. Formulas will be discussed in later, See Section 3.8, Formula Screen.. Below the system status area is displayed the current date and time (5). The APM software user account under which the user is currently logged in is displayed (6), and the Wenger shop order number that was assigned to the customer's extruder system (7) is always displayed.

3.1.2.2 Main Screen Header - Mid Zone

The mid region of the header (see Figure 3.3) displays several items unrelated to each other. The extruder load (1), displayed at the top in a light blue background color, is a measure of the extruder motor's energy consumption expressed as a percent of full load. By clicking the (Home) Main button (3), in a gray background color, the user can quickly return to the Main screen from any other APM screen. Between the extruder load display and the (Home) Main button is a space (2), shown empty in Figure 3.1, where “Emergency Stop” text will appear in a red foreground color when the emergency stop button has been pressed but before it has been reset.

3.1.2.3 Main Screen Header - Right Zone

The right side of the header displays the three most current extruder system alarms (see Figure 3.4).

All alarms that have arisen and displayed after the most recent alarms acknowledgment by the user will be displayed in red foreground color (as seen in Figure 3.4). All acknowledged alarms appear in yellow foreground color.

3.1.2.4 Status Bar

The APM status bar at the bottom of the screen displays diagnostic information pertaining to the behavior of the APM software. It should not be confused with the extruder system status information displayed in the header. The information displayed in the APM status bar will probably be of more interest to Wenger Technicians than to customer personnel.

3.1.3 Graphical Shapes and Cooler Conventions

3.1.3.1 Devices

The various types of components that make up the extruder system, such as motors, pumps, valves, and fans, are depicted graphically on a number of APM screens. All screens that display the status of such equipment (e.g., whether motors are running or stopped) use a common color scheme to quickly communicate equipment status to the operator, regardless of the type of equipment.

The status for valves (i.e., open or closed) is communicated by the fill color of each respective valve's graphical depiction. Referring to Figure 3.5, an open valve is depicted with a green color (1), while a closed valve is depicted with a blue color(2).

Referring to Figure 3.6, a running motor is depicted with a green colored image (1), while a stopped motor is depicted with a blue colored image (2). In addition, the graphics for some types of devices, such as motors, are accompanied by ellipses, wherein the status will be displayed with text, in the case below, either On or Off.

Each status ellipse, in addition to displaying text that communicates the running or not running status of the corresponding device motor, also displays the control mode (Hand, Off or Auto) that the PLC software recognizes the device to be in, indicated by the background color of the ellipse. A blue background color signifies that the PLC considers the device to be in the Off control mode, white signifies the device is in Auto control mode, and yellow signifies the device is in Hand control mode. Lastly, a red ellipse background color that alternates with blue, white or yellow indicates that the device in question has encountered some type of fault.

So, for example, referring to Figure 3.7, the full status of the extruder lube pump is:

  1. Not currently running, and in the Off control mode.
  2. Not currently running, and in the Auto control mode.
  3. Running, and in the Auto control mode.
  4. Running, and in the Hand control mode
  5. Not currently running because of a fault condition

3.1.3.2 Process Flows

Three types of process flows are depicted graphically with colors on APM screens: ingredients and product (this includes raw feed, oil, color, fat, etc.), process water, and process steam. (Note that water and steam are also used in the extruder temperature control jackets, but these are considered temperature control elements rather than process flows.) Colors only indicate the binary status (path is open for flow vs path is closed to flow) of the equipment through which material normally travels.

For ingredients and product, just as for equipment, green and blue are the two colors used to indicate binary flow status, green signifying the path that is open to product flow, and blue signifying that the flow through the equipment in question is prevented. In Figure 3.8, the green color (1) of the conduit between the knife hood diverter and the dryer indicates that product can flow, but notice from the blue color (2) immediately below the knife hood diverter that product is prevented from flowing up through the knife hood.

For process water, light blue and black are the two colors used to indicate binary flow status, light blue signifying that process water can flow, and black signifying that process water cannot flow. In Figure 3.9, notice how process water is being injected into the extruder at 15 kg/hr (1), but that no process water is being injected into the preconditioner above (2).

For process steam, white and black are the two colors used to indicate binary flow status, white signifying that process steam can flow, and black signifying that process steam cannot flow. In Figure 3.9, notice that process steam is being injected into the preconditioner at 25 kg/hr (3), but that no process steam is being injected into the extruder (4).

3.1.4 Hand-Off-Auto and Proportional-Integral-Derivative Faceplate’s

A Hand-Off-Auto (HOA) or Proportional-Integral-Derivative (PID) faceplate is a pop-up window (i.e., small faceplate) that appear when the operator clicks an HOA or PID button pertaining to a specific device. Each HOA faceplate presents the operator with options that he can choose that determine how the corresponding extruder system device or condition is controlled by the PLC. There is an HOA faceplate for each device and a PID faceplate for each process flow or temperature that the PLC controls. Most of these faceplate’s are accessible from the Extruder Overview screen. (See Section 3.3, Extruder Overview Screen.) Other faceplate’s are accessible from the Extruder Barrel Temperatures screen (See Section 3.4, Extruder Barrel Temperatures Screen.), and from the Extruder Liquids 1 and Extruder Liquids 2 screens (See Section 3.5, Extruder Liquids 1 Screen. and See Section 3.6, Extruder Liquids 2 Screen., respectively).

The options that a faceplate presents to the operator for device or condition control depend on the nature of the device or condition being controlled. There are devices that are driven by fixed-speed motors, so the HOA faceplate’s for these will use the same template and will offer the same control options. There are also variables speed motors, so the HOA faceplate’s for these will offer the same control options. There are discrete position gates for handling the flow of solid materials, so the HOA faceplate’s for these will offer the similar control options. There are proportional control valves for controlling the flow of liquids and vapors using flow PID control algorithms, all of which use a common PID faceplate template. Lastly, there are proportional control valves for controlling the flow of steam and cooling water using temperature PID control algorithms, all of which use a common PID faceplate template. The faceplate’s described above are listed below in approximate order of increasing complexity:

  • HOA faceplate’s for motors driving fixed-speed devices.
  • HOA faceplate’s for discrete-position solid material handling gates.
  • HOA faceplate’s for motors driving variable-speed devices.
  • PID faceplate’s for proportional position valve for controlling liquid and vapor flow rates
  • PID faceplate’s for proportional position valves for controlling temperatures.

3.1.4.1 HOA Faceplate’s for Motors Driving Fixed-Speed Devices

The HOA for motors that drive fixed-speed devices are the simplest HOA to understand, so this type of HOA faceplate will be addressed first. Most fixed-speed motors are unidirectional, but some are bidirectional. Examples of devices driven by fixed-speed motors are:

  • Live Bin Agitators
  • Certain Types of Preconditioners
  • Extruder Lube Oil Pumps
  • Liquid Tank Agitators

The only parameter for a fixed-speed device motor that an operator can change using its HOA faceplate is the device's control mode. Using the HOA faceplate for a fixed-speed motor, the operator can change its control mode to either Hand, Off, or Auto.

Hand Control Mode - Setting a fixed-speed device motor to Hand control mode instructs the PLC that the operator is commanding the motor to run, overriding the PLC's Auto control mode logic.

Off Control Mode - Setting a fixed-speed device motor to Off control mode instructs the PLC that the operator is commanding the motor to stop, overriding the PLC's Auto control mode logic.

Auto Control Mode - Setting a fixed-speed device motor to Auto control mode instructs the PLC that the decision whether the device should run or stop is to be made by the PLC's Auto control mode logic (algorithms programmed into the PLC that the PLC uses to decide at any time whether to run the device; See Section 3.15, Display Languages switching. for a discussion of these algorithms.

The PLC attempts to prevent any device from running if there are active conditions for which running the device might cause injury to personnel or damage to equipment. These potential unsafe conditions vary with each device. However, the extruder system operator is ultimately responsible for ensuring that the extruder system is safe to run.

3.1.4.2 HOA Faceplate’s for Motors Driving Variable-Speed Devices

Since the HOA for devices driven by variable-speed motors are similar to HOA for devices driven by fixed-speed motors, this type of HOA faceplate will be addressed next. Examples of devices driven by variable-speed motors are:

  • Extruder's
  • Feeders
  • Certain Types of Preconditioners
  • Knives
  • Liquid Pumps

There are two parameters for a variable-speed device motor that an operator can change using its HOA faceplate, these being the device's control mode and its speed. The motor speed can only be changed by the operator using the HOA faceplate when the device is in Hand control mode. When the device is in Auto control mode, the device speed is determined from formula parameter values. Using the HOA faceplate for a variable-speed motor, the operator can change its control mode to either Hand, Off, or Auto.

Hand Control Mode - Setting a variable-speed device motor to Hand control mode instructs the PLC that the operator is commanding the motor to run, overriding the PLC's Auto control mode logic. When a variable-speed device is in Hand control mode the device speed is determined from its manual setpoint, expressed as a percentage that ranges from 0% (the minimum speed that the motor is capable of) to 100% (the maximum speed that the motor is capable of). Using the HOA faceplate the operator can enter the manual setpoint directly as an integer value, or he can ramp the setpoint up and down using an up arrow button or down arrow button, respectively. The faceplate objects for specifying the device's manual setpoint only appear on the HOA faceplate when the Hand control mode has been selected.

Off Control Mode - Setting a variable-speed device motor to Off control mode instructs the PLC that the operator is commanding the motor to stop, overriding the PLC's Auto control mode logic (no different than for fixed-speed devices).

Auto Control Mode - Setting a variable-speed device motor to Auto control mode instructs the PLC that the decision whether the device should run or stop is to be made by the PLC's Auto control mode logic (algorithms programmed into the PLC that the PLC uses to decide at any time whether to run the device, See Section 3.15, Display Languages switching. for a discussion of these algorithms). The device's Auto control mode speed setpoint is determined by the PLC, usually from a corresponding formula parameter value. The speed setpoints for some devices, such as the extruder and knife, can be changed by the operator on other APM screens. For other devices, such as the feeder screw and the liquid pumps, the speed setpoints are calculated by the PLC from other parameters, such as the desired dry feed rate. Regardless, Auto control mode speed setpoints cannot be changed using HOA faceplate’s.

The PLC monitors for certain unsafe conditions, and if it determines that any of these are active, the PLC attempts to keep the device in the Stopped state, even if the device is in the Hand control mode.

The PLC attempts to prevent any device from running if there are active conditions for which running the device might cause injury to personnel or damage to equipment. These potential unsafe conditions vary with each device. However, the extruder system operator is ultimately responsible for ensuring that the extruder system is safe to run.

3.1.4.3 HOA Faceplate’s for Gates

There are several gates that are part of the extruder system. These gates are discrete-position devices, meaning that they can be commanded to either of two or three positions. Using the HOA faceplate for the corresponding gate, the operator can change the device's control mode, and when in either the Hand or Off control mode, the operator can set the gate's discrete position. These discrete positions are named differently on the HOA faceplate’s, depending on the gate. These devices are:

  • Dry Feed Slide Gates
  • Downspouts
  • Back Pressure Valves (BPV)
  • Knifehood Diverters

The dry feed slide gate is a two-position gate situated between the surge bin and the live bin, and is used to refill the live bin with dry feed from the surge bin. The slide gate has two discrete positions: completely open, which allows dry feed to flow by gravity from the surge bin into the live bin, and completely closed, which allows no dry feed to flow. Thus, the two positions for the slide gate are named Open and Closed.

The downspout is a two-position gate situated between the preconditioner and the extruder and is used to channel feed mixture that flows by gravity out of the preconditioner into the extruder inlet. The downspout has two discrete positions: one position channels the feed mixture from the preconditioner into the inlet of the extruder, while the other position channels feed mixture from the preconditioner out of the extruder system altogether (in which case it must be collected somehow or it will pile up on the production facility floor). Thus, the two positions for the downspout are called Extruder and Bypass.

The knifehood diverter is a two-position gate situated between the top of the knifehood and the inlet of the processing equipment downstream of the extruder system, and is used to allow extrudate slices to be sucked into the downstream processing system, via negative pressure created by the downstream pneumatic product transfer equipment. The knifehood diverter has two positions: completely open, which allows raw product to travel downstream for further processing, and completely closed, in which case the extrudate slices are rejected and fall out of the bottom of the knifehood (in which case they must be collected somehow or they will pile up on the production facility floor). Thus, the two positions for the knifehood diverter are called Process and Reject.

Using the HOA faceplate for gate device, the operator can change its control mode to either Hand, Off, or Auto.

Hand Control Mode - Setting a gate device to Hand control mode instructs the PLC that the operator is commanding the gate position, overriding the PLC's Auto control mode logic. Which of the discrete gate positions corresponds to the Hand control mode depends on the gate: Open for the slide gate, Extruder for the downspout, and Process for the Knifehood diverter.

Off Control Mode - Setting a gate device to Off control mode instructs the PLC that the operator is commanding the gate position, overriding the PLC's Auto control mode logic. Which of the discrete gate positions corresponds to the Off control mode depends on the gate: Closed for the slide gate, Bypass for the downspout, and Reject for the Knifehood diverter.

Auto Control Mode - Setting a gate device to Auto control mode instructs the PLC that the decision whether the device should run or stop is to be made by the PLC's Auto control mode logic (algorithms programmed into the PLC that the PLC uses to decide at any time whether to run the device; See Section 3.6, Extruder Liquids 2 Screen. for a discussion of these algorithms).

The PLC monitors for certain unsafe conditions, and if it determines that any of these are active, the PLC attempts to keep the device in the Stopped state, even if the device is in the Hand control mode.

The PLC attempts to prevent any device from running if there are active conditions for which running the device might cause injury to personnel or damage to equipment. These potential unsafe conditions vary with each device. However, the extruder system operator is ultimately responsible for ensuring that the extruder system is safe to run.

3.1.4.4 PID Faceplate’s for Controlling Flow Rates

A flow rate PID control loop, when placed in Auto control mode, maintains a constant liquid or vapor flow to match the desired flow rate (i.e., the control loop's Auto control mode setpoint) either by varying a control valve position (as in the case for process water and steam injected into the preconditioner and extruder), or by varying a metering pump speed (as in the case of most customer-specific liquids such as oils, dyes, flavorings). This same flow rate PID control loop, when placed in Manual control mode, maintains a constant control valve position or metering pump speed (i.e., the control loop's Manual control mode setpoint) at the expense of a constant liquid or vapor flow rate through the valve or pump, which may vary with other conditions, such as upstream and downstream pressures.

There are a number of flow rate PID control loops that are used to control extruder system flows: These flows are:

  • Process water to the preconditioner.
  • Process steam to the preconditioner.
  • Customer specific liquids to the preconditioner.
  • Process water to the extruder.
  • Process steam to the extruder.

There are three parameters for a flow PID control loop that an operator can change using its PID faceplate, these being the control loop's control mode, the flow rate setpoint (the Auto control mode setpoint), and the control valve position or metering pump speed (the Manual control mode setpoint). The flow rate setpoint can only be changed using the PID faceplate when the PID control loop is in Auto control mode, and the control valve position or metering pump speed can only be changed using the PID faceplate when the control loop is in Manual control mode. Using the PID faceplate the operator can switch between Auto and Manual control modes. There is no Off control mode.

Manual Control Mode - Setting a flow PID control loop to Manual control mode instructs the PLC to maintain a constant control valve position (the degree to which it is open) or metering pump speed. This desired valve position or pump speed is the manual setpoint, expressed as a percentage that ranges from 0% (the valve is completely closed or the pump is stopped) to 100% (the valve is completely open or the pump is running at its maximum speed). Using the PID faceplate, the operator can enter the manual setpoint directly as a numerical value, or he can ramp the setpoint up and down using the up or down arrow buttons, respectively. The means of specifying the device's manual setpoint only appear on the PID faceplate when the Manual control mode has been selected.

Auto Control Mode - Setting a flow PID control loop to Auto control mode instructs the PLC to maintain a constant flow rate of material, specifically mass flow rate. This desired mass flow rate is calculated from the auto setpoint, which is expressed as a percentage by weight of the dry feed rate. Thus, if the extruder steam auto setpoint is specified as 10% of feed rate, and the dry feed rate is 1000 kg/hr, the extruder steam control loop will attempt to maintain the extruder steam flow rate of 100 kg/hr. Using the PID faceplate, the operator can enter the auto setpoint directly as a numerical value, or he can ramp the setpoint up and down using up or down arrow buttons, respectively. The means of specifying the device's auto setpoint only appear on the PID faceplate when the Auto control mode has been selected.

3.1.4.5 PID Faceplate’s for Controlling Temperatures

The PID faceplate’s for controlling temperatures are similar to the PID faceplate’s for controlling flow rates. However, for two reasons, controlling extruder barrel temperatures with PID control loops is somewhat more complex than controlling flow rates with PID control loops. First, controlling the temperature of an extruder barrel zone requires moderating both the flow of water for cooling and the flow of steam for heating. Thus, varying the positions of two valves are necessary in temperature control, whereas varying the position of only one valve or the speed of one pump is required in flow control. Second, the valves for controlling steam or cooling water to the extruder barrel jackets are on-off valves rather than proportional-position control valves. For each of these valves, what is ultimately controlled is not valve position but rather the ratio of time over a fixed time period, say ten seconds, that each valve is completely open vs completely closed.

A temperature PID control loop, when placed in Auto control mode, attempts to maintain a temperature to match the desired temperature (i.e., the control loop's Auto control setpoint). When in Auto control mode and the actual temperature is below the desired temperature, the PLC closes completely the cooling water valve while the PLC varies the length of time during a fixed length time period that the steam valve is completely open. When in Auto control mode and the actual temperature is above the desired temperature, the PLC closes completely the steam valve while the PLC varies the length of time during a fixed length time period that the cooling water valve is completely open. This same temperature PID control loop, when placed in Manual control mode, maintains constant time-based ratios for the each of the two valves (derived from the control loop's Manual control setpoint), thus allowing the temperature to vary with other conditions, such as steam and cooling water upstream pressures or production rate changes.

There are a number of temperature PID control loops that are used to control extruder system temperatures, depending on the number of extruder barrel temperature zones that were provided with the customer's extruder system.

There are three parameters for a temperature PID control loop that an operator can change using its PID faceplate, these being the control loops' control mode, the temperature setpoint (the Auto control mode setpoint) and the PID control loop output (the Manual control mode setpoint). The temperature setpoint can only be changed using the PID faceplate when the PID control loop is in Auto control mode, and the PID control loop output can only be changed using the PID faceplate when the control loop is in Manual control mode. Using the PID faceplate the operator can switch between Auto and Manual control modes. There is no Off control mode.

Manual Control Mode - Setting a temperature PID control loop to Manual control mode instructs the PLC to maintain constant time-based ratios that the two control valves are open or closed. This desired PID control loop output is the manual setpoint, expressed as a percentage that ranges from 0% (maximum cooling) to 100% (maximum heating). A 50% manual setpoint corresponds to no heating or cooling. Thus, when the manual setpoint is 0%, the cooling water valve is open 100% of the time while the steam valve remains completely closed. When the manual setpoint is 25%, the cooling water valve is open 50% of the time while the steam valve remains completely closed. When the manual setpoint is 50%, both the cooling water valve and the steam valve remain completely closed. When the manual setpoint is 75%, the steam valve is open 50% of the time while the cooling water valve remains completely closed. And finally, when the manual setpoint is 100%, the steam valve is open 100% of the time while the cooling water valve remains completely closed. Using the PID faceplate, the operator can enter the manual setpoint directly as a numerical value, or he can ramp the setpoint up and down using the up or down arrow buttons, respectively. The means of specifying the device's manual setpoint only appear on the PID faceplate when the Manual control mode has been selected.

Auto Control Mode - Setting a flow PID control loop to Auto control mode instructs the PLC to maintain a constant zone temperature. Using the PID faceplate, the operator can enter the auto setpoint directly as a numerical value, or he can ramp the setpoint up and down using up or down arrow buttons, respectively. The means of specifying the device's auto setpoint only appear on the PID faceplate when the Auto control mode has been selected.

The temperature of the extruder barrel's temperature zone 1 will be used as an example in this discussion. The PID button (1) for control of extruder barrel zone 1 temperature is shown in Figure 3.25. Clicking this button causes the zone 1 temperature PID faceplate to appear (see Figure 3.26).

  2. Auto Control Mode Temperature Setpoint Select - This group of controls allow the operator to change the PID control loop's temperature setpoint, ramping up or down by clicking the arrow buttons, or entering a numerical value by clicking inside the white rectangular field. When the displayed output value is between 0% and 50%, the percentage of time that the cooling water valve is open is between 100% open and 0% open, respectively. At the same time the heating steam valve is completely closed whenever the output display is below 50%. When the displayed output value is between 50% and 100%, the percentage of time that the heating steam valve is open is between 0% open and 100% open, respectively. At the same time the cooling water valve is completely closed whenever the output display is above 50%. Therefore, when the displayed output value is exactly 50% neither heating nor cooling is being commanded, and thus both steam and water valves are completely closed. When the displayed output value is exactly 0% maximum cooling is being commanded, and thus the water valve is completely open while the steam valve is completely closed. And when the displayed output value is exactly 100% maximum heating is being commanded, and thus the steam valve is completely open while the water valve is completely closed.
