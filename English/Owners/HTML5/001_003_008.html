3  APM Screens

3.8 Formula Screen

3.8.1 Section Overview

Formulas, also referred to as recipes, are collections of extruder system parameters (e.g., operating condition setpoints, time intervals between the events in a sequence of events, specifiers that determine how the extruder system is to be started up) that a customer has determined are necessary to formulate the various products in the customer's product line. The number of parameters that must be set to formulate each product can be quite large. Thus, it is desirable to be able to specify each collection of values just once, and store this collection of values in a formula file on the APM hard drive. Thereafter, when it is necessary to formulate a new batch of that product, the necessary extruder system parameters can be retrieved from the corresponding formula file on the APM computer's hard drive and downloaded to the PLC without the operator needing to manual specify them again. This saves time and minimizes the possibility of human error.

The operator will manage formulas by using three screens, the Formula screen and the Load Formula screen and the Edit/Add Formula screen, along with a handful of small faceplates. Using these tools, the operator will be able to perform the following formula-specific tasks:

  • Create completely new formulas.
  • Create new formulas from existing formulas.
  • Modify existing formulas.
  • Delete existing formulas.
  • Create new formulas from current PLC setpoints.
  • Updating formula values with PLC setpoints.
  • Transfer formula values to the PLC to be used as the new setpoints.

The formula data that is stored on the APM computer's hard drive are stored in Microsoft SQL Server database files.

The Formula screen is accessed by clicking the (F8) Formula button in the region of the Main screen with the forest green background color. It can also be accessed from any screen by pressing the F8 function key on the APM console keyboard. The Load Formula screen is accessed from the Formula screen. The Edit/Add Formula screen is accessed from the Load Formula screen.

3.8.2 Formula Screen

An example of the Formula screen is shown in Figure 3.74. There are three collections of values displayed on this screen:

  • The selected formula data that have been retrieved from a selected SQL database formula stored on the APM computer's hard drive. This data is displayed in the two columns of data under the “Formula Value” headers, and in the top left region of the screen with the dark blue background color, under the header labeled “Selected Formula”. From the Formula screen, this data can be loaded into the PLC's memory, thus overwriting the PLC's current setpoint values.
  • The setpoint data that reside in the PLC's memory and are currently being used by the PLC to control the extruder system. This data is displayed in the two columns of data under the “PLC Setpoint” headers, and in the top right region of the screen with the dark blue background color, under the header labeled “PLC”. From the Formula screen, this data can be saved to an existing or new formula database file on the APM computer's hard drive, or overwritten with the selected formula data.
  • Current operating conditions that result from the setpoint data used by the PLC to control the extruder system. This constantly updated information is obtained by the PLC from the various input/output modules in the extruder system control panel, and is displayed in the two columns of data under the “Actual Value” headers. Not all formula parameters are setpoints, directly impacting current operating conditions. Thus, some of the cells in the “Actual Value” columns are blank. This data is for information only and is not stored in either a formula database file or in PLC memory.
  1. Formula Data Retrieved from Database - This block of data provides general information about the selected formula database file on the APM computer's hard drive. This information includes the formula database file name, a short description of the formula, revision number and date, and the extruder die number.
  2. Formula Data in PLC Memory - This block of data provides general information about the formula parameters that reside in PLC memory and that the PLC is currently using to control the extruder system. This information is related to the formula database file on the APM computer's hard drive from which the data was retrieved before being loaded into the PLC's memory. This data includes the formula database file name, a short description of the formula, revision number and date, and the extruder die number.
  3. Formula Select/Maintenance Button - Clicking this button causes the Load Formula screen to appear (more about this later).
  4. Load Formula to PLC Setpoints Button - Clicking this button copies into PLC memory the currently selected formula parameters that have been retrieved from an APM computer database file (and possibly edited, more about this later), and causes the PLC to use these new values as the setpoints with which to control the extruder system.
  5. Save PLC Setpoint Buttons - Clicking this button brings forth a small faceplate with which the operator may use to overwrite the data in the currently selected formula database file stored an APM computer's hard drive with the setpoints currently residing in PLC memory.
  6. Item Description Columns - These two columns of text display short descriptions of the purpose of each pair of formula parameter and PLC setpoint.
  7. Formula Value Columns - These two columns display the values for the various formula parameters that have been retrieved from a formula database file and possibly edited.
  8. PLC Setpoint Columns - These two columns display the values for the setpoints that reside in PLC memory and currently used by the PLC to control the extruder system.
  9. Actual Value Columns - These two columns display actual extruder system operating conditions resulting from the PLC's use of its setpoints to control of the extruder system.

3.8.3 Load Formula Screen

An example of the Load Formula screen is shown in Figure 3.75. This screen is accessed when the Formula Select/Maintenance button on the Formula screen is clicked. The Load Formula screen displays all formula database files currently residing on the APM computer's hard drive. This screen also provides the operator with the means of filtering the display of existing formula database files using either file name or formula description as the filter criterion. Finally, this screen allows the operator to perform the following tasks:

  • Select an existing formula database file (causes the formula data to be read from the chosen database file and displayed on the Formula screen as the selected formula).
  • Edit an existing formula database file (causes the formula data to be read from the chosen database file and displayed on the Add/Edit Formula screen that appears in place of the Load Formula screen).
  • Create a completely new formula database file (causes the Add/Edit Formula screen to appear with all parameter fields filled with the new formula default values.
  • Delete an existing formula database file (causes the chosen database file to be deleted).
  1. List of Current Formulas (Forms Column) - This column in the list box displays the names of all formulas currently stored on the APM computer's hard drive.
  2. List of Current Formulas (Rev Column) - This column in the list box displays the revision number for each of the formulas currently stored on the APM computer's hard drive. Multiple formulas may be stored with the same name provided they have different revisions.
  3. List of Current Formula (Description Column) - This column in the list box displays the short textual description for each of the formulas currently stored on the APM computer's hard drive.
  4. “Select Formula” Button - This button is disabled until a formula in the list box to the left is clicked. Then, when this button becomes enabled and when clicked, the formula data for the chosen formula is retrieved from the formula database file on the APM computer's hard drive and displayed as the selected formula parameters on the Formula screen.
  5. “Edit Formula” Button - This button is disabled until a formula in the list box to the left is clicked. Then, when this button becomes enabled and when clicked, the formula data for the chosen formula is retrieved from the formula database file on the APM computer's hard drive and displayed as the formula parameters ready to be edited on the Add/Edit Formula screen.
  6. “Add Formula” Button - This button is always enabled, and when clicked, causes the Add/Edit Formula screen to appear with all its parameter fields filled with the formula default values.
  7. “Delete Formula” Button - This button is disabled until a formula in the list box to the left is clicked. Then, when this button becomes enabled and when clicked, the chosen formula is deleted from the formula database on the APM computer's hard drive.
  8. “Refresh” Button - This button is always enabled, and when clicked, refreshes the information displayed in the list box to the left using the search criterion specified with the Search By and Search Options filter tools.
  9. “Close” Button - This button is always enabled, and when clicked, closes the Load formula screen.
  10. Search By Tool - Allows the operator to search for formulas by text in their description fields rather than by formula name.
  11. Search Options Tool - Allows the operator to refine his formula description search filter criterion.

3.8.4 Add/Edit Formula Screen

An example of the Add/Edit Formula screen is shown in Figure 3.76. This screen is accessed when either of the Edit Formula or Add Formula buttons on the Load Formula screen is clicked. This screen is also accessed when the Save PLC Setpoints button on the Formula screen is clicked, and then followed by the operator clicking either of the two Save buttons on the Update Anyway faceplate.

When the Add/Edit Formula screen appears as a result of the Edit Formula button on the Load Formula screen being clicked, all fields are populated with values from the most recently selected formula. When the Add/Edit Formula screen appears as a result of the Add Formula button on the Load Formula screen being clicked, all fields are populated with the formula default values. When the Add/Edit Formula screen appears as a result of the Save PLC Setpoint button on the Formula screen being clicked, all fields are populated with values currently residing in PLC memory.

Regardless, once the Add/Edit formula screen appears the operator can change any of the displayed values, for example specifying a new formula revision, even a new formula name, or changing any of the individual formula parameter values. Then he can overwrite the data in the existing formula in the formula database on the APM computer's hard drive, save the data as a new formula in the database, or he can cancel without making any changes to the formula database.

  1. Formula No. - This is the formula identifier for the currently selected formula as it is stored in the formula database. It can be either a number or text. The operator can create a new formula by clicking in the white rectangle, in which case a faceplate will appear prompting him to enter a new formula identifier.
  2. Revision - This is the revision number for the currently selected formula. Each formula in the database can have multiple revisions, and they are each maintained separately. The operator can create a new revision for the formula by clicking in the white rectangle, in which case a faceplate will appear prompting him to enter a new formula revision number.
  3. Last Revision Date - This is date when this revision of the formula was most recently updated.
  4. Description - This is a field where the operator can provide a short description of what the formula is intended to be used. The operator can edit this field by clicking in the white rectangle, in which case a faceplate will appear prompting him to enter a new description.
  5. Individual Formula Parameters - The operator can edit the values in any of these field by clicking in the white rectangle, in which case a faceplate will appear prompting him to enter a new parameter value.
  6. “Update Formula” Button - Clicking this button will cause a small faceplate to appear from which from operator can confirm that he wants to update, update and select, or cancel.
  7. “Cancel” Button - Clicking this button will cause a small faceplate to appear prompting the operator to confirm that he wants to cancel (in which case the Add/Edit Formula screen will close without saving any edits) discontinue canceling (in which case the Add/Edit Formula screen will remain open and allow the operator to resume editing).
  8. “Abort” Button - Clicking this button will cause the Add/Edit formula screen to immediately close (in which case any edits will be discarded).

3.8.5 Creating a Completely New Formula

To create a new completely new formula, first display the Formula screen, and on it click the Formula Select/Maintenance button (1) (see Figure 3.77).

Next, when the Load Formula screen appears, click the Add Formula button (1) (see Figure 3.78).

The Add/Edit Formula screen (see Figure 3.76) will appear with all fields populated with the default formula values. The only value that absolutely must be changed from its default value before the formula can be updated is the value in the Formula No field. Click inside the white rectangles to edit the various formula values. When finished editing, click the Update Formula button (1) on the Add/Edit Formula screen (see Figure 3.79).

The Add Formula? faceplate will appear, as shown in Figure 3.80, where the formula identifier (1) and the formula revision (2) are displayed. To store the newly defined formula to the formula database on the APM computer hard drive, click the Yes button (3). This will only store, but not select (i.e., retrieve) the formula values from the new stored formula. The Formula screen (see Figure 3.74) will reappear, but the new formula will not be the selected formula. To both store the new formula and then retrieve its values as the selected formula on the Formula screen, click the Yes and Select button (4) in Figure 3.80. To return to the Formula Edit screen click the Cancel button (5) in Figure 3.80.

If the Update Formula button on the Formula Add/Edit screen is clicked with the value in the Formula No field unchanged from its default value of 0, the Default Formula faceplate in Figure 3.81 will appear, prompting the operator to change the value in the Formula No field. Clicking the OK button (1) will return the operator to the Add/Edit Formula screen where the Formula No field can be changed.

3.8.6 Creating a New Formula from an Existing Formula

To create a new formula from an existing formula, first display the Formula screen, and on it click the Formula Select/Maintenance button (1) (see Figure 3.82).

Next, when the Formula Load screen appears, choose an existing formula that has values as similar as possible to those desired for the new formula (1), and click the Edit Formula button (2), as shown in Figure 3.83.

The Add/Edit Formula screen (see Figure 3.76) will appear with all fields populated with values from the existing chosen formula. The only value that absolutely must be changed from its default value before the formula can be updated is the value in the Formula No field. Click inside the white rectangles to edit the various formula values. The only values that must be changed from the values read from the existing formula are the values in the Formula No field and/or the Revision field. A new formula can be created by changing either of these fields, but it is wisest to change both. When finished editing, click the Update Formula button (1) on the Add/Edit Formula screen (see Figure 3.84).

The Add Formula? faceplate will appear, as shown in Figure 3.85, where the formula identifier (1) and the formula revision (2) are displayed. To store the newly defined formula to the formula database on the APM computer hard drive, click the Yes button (3). To store and select the new formula click the Yes and Select button (4). To return to the Formula Edit screen click the Cancel button (5).

  1. “Flush Extruder” Button - Clicking this button commands the PLC to execute the extruder flush utility. Refer to Extruder Barrel Flush Sequence for a detailed description of how this utility is used.
  2. “Stop Flushing Extruder” Button - Clicking this button commands the PLC to terminate the extruder flush utility. Refer to Extruder Barrel Flush Sequence for a detailed description of how this utility is used.
  3. “Normal Flush Cylinder” Button - Clicking this button commands the PLC to execute the preconditioner normal flush utility. Refer to Preconditioner Flush Sequences for a detailed description of how this utility is used.
  4. “Wet Flush Cylinder” Button - Clicking this button commands the PLC to execute the preconditioner wet flush utility. Refer to Preconditioner Flush Sequences for a detailed description of how this utility is used.
  5. “Stop Flushing Cylinder” Button - Clicking this button commands the PLC to terminate whichever preconditioner flush utility is being executed by the PLC. Refer to Preconditioner Flush Sequences for a detailed description of how this utility is used.

3.8.7 Modifying an Existing Formula

To modify an existing formula, first display the Formula screen, and on it click the Formula Select/Maintenance button (1) (see Figure 3.86).

Next, when the Formula Load screen appears, choose the formula to modify (1), and click the Edit Formula button (2), as shown in Figure 3.87.

The Add/Edit Formula screen (see Figure 3.76) will appear with all fields populated with values from the chosen formula. Click inside the white rectangles to edit the various formula values. Do NOT change the “Formula No.” value or the “Revision” value. When finished editing, click the Update Formula button (1) on the Add/Edit Formula screen (see Figure 3.88).

The Add Formula? faceplate will appear, as shown in Figure 3.89.

  1. “Update Formula” Button - Click this button when finished editing, on the “Add/Edit Formula” screen (see Figure 3.88).
  1. Formula: - This displays the formula identifier.
  2. Revision: - This displays the formula revision.
  3. “Yes” Button - Clicking this button to store the modified formula to the formula database on the APM computer hard drive.
  4. “Yes and Select” Button - Clicking this button to store and select the new formula.
  5. “Cancel” Button - Clicking this button to return to the Formula Edit screen.

3.8.8 Deleting an Existing Formula

To delete an existing formula, first display the Formula screen, as shown in Figure 3.90).

  1. “Formula Select/Maintenance” Button - Click this button to bring up the formula load screen.

Next, when the Formula Load screen appears as shown in Figure 3.91.

  1. Formula - Choose the formula to delete.
  2. “Delete Formula” Button - Click this button to delete the formula.

The Delete? faceplate in Figure 3.92 will appear.

  1. “Yes” Button - Click this button to delete the formula.
  2. “No” Button - Click this button to return to the Formula screen without deleting the formula.

3.8.9 Updating Formula Values with Current PLC Setpoints

This operation involves overwriting the formula values in the currently selected formula with the setpoints in PLC memory. Before starting this operation, the currently selected formula should be one that has values that the operator wants to replace with the PLC setpoints.

To update the values of the currently selected formula with the setpoints in PLC memory, first display the Formula screen shown in Figure 3.93.

  1. “Save PLC Setpoints” Button - To save updated formula values in the PLC memory.

The faceplate shown in Figure 3.94 appears, warning the operator of a conflict, specifically that he is trying to save data to a database formula that already exists for the given formula name and revision.

  1. Formula: - This shows the formula name that already exists, that was trying to be saved to the database.
  2. Revision: - This shows the revision number that already exists, that was trying to be saved to the database.
  3. “Save Extruder Setpoints Only” Button - Clicking this button will save ONLY the extruder setpoints.
  4. “Save Extruder & Downstream Equip. Setpoint” Button - Clicking this button will save the extruder & downstream equipment setpoint.
  5. “Cancel” Button - Clicking this button can discard the formula data uploaded from the PLC.
