2 APM Startup

2.1 Overview

This chapter addresses what the operator will need to know to get the APM computer up and running and to the point that the APM software is also running. It describes the different Windows and APM software user accounts that exist, how to boot up the APM computer, how to get the APM application software running, how to switch among the different levels of APM user types, and for APM systems that were developed for customers in Non-English speaking CE countries, how to switch the language that the APM software uses to display text between English and the customer's local language.

2.1.1 User Accounts

For the reader to avoid being confused by the following sub-paragraphs, he should understand that two types of user accounts will be discussed. The first type are the Windows operating system user accounts, of which there are two, and the second type are the APM software user accounts, of which there are four. Note that the names of the two Windows user accounts are also used as names for two of the four APM user accounts.

2.1.2 Windows Operating System User Accounts

On every APM computer that is delivered to a customer, Wenger has created the following two user accounts at the Windows operating system level:

  • Operator
  • Wenger

The “Wenger” Windows user account is password-protected but the “Operator” Windows user account is not. Only Wenger personnel know the “Wenger” Windows user account password, so no non-Wenger personnel will be able to login to Windows using the “Wenger” user account. It will be necessary for all customer personnel, regardless of whether they are operators, maintenance personnel, or supervisors, to login to Windows using the “Operator” Windows user account, but since this Windows user account is not password-protected, the user will not be prompted for a password.

2.1.3 APM Software User Accounts

At the APM application software level, Wenger has created the following four user accounts:

  • Operator
  • Maintenance
  • Supervisor
  • Wenger

These four APM software user accounts provide the users with differing levels of APM software privileges. The “Operator” APM user type is the most restrictive. The “Maintenance” APM user account is slightly less restrictive. The “Supervisor” APM user account is much less restrictive. And finally, the “Wenger” APM user account has no restrictions associated with it. Note that the Maintenance, Supervisor and Wenger user accounts at the APM software level are password protected, while the Operator user account is not password-protected.

Due to the potential for confusion, it will be stressed here again. The “Operator” and “Wenger” user accounts that have been defined at the Windows operating system level should not be confused with the “Operator”, “Maintenance”, “Supervisor” and “Wenger” user accounts that have been defined at the APM application software level.

All customer personnel must login to Windows using the “Operator” Windows user account. When this happens, and after the APM software starts up, the APM software automatically logs the customer into the APM software using the “Operator” APM application software's Operator user account. But using the APM software, customer personnel can switch to either of the less restrictive Maintenance or Supervisor types (if they know the respective user type passwords). This will be described in detail below

2.1.4 Powering Up the Computer, and Logging into Windows

When the HMI computer is first powered up it boots up with whatever version of the Windows operating system has been installed on the computer. When bootup is complete, Windows prompts the user to login to Windows. There are two types of Windows user accounts that are available for selection, “Operator” and “Wenger” (see Figure 2.1).

  1. Operator - Clicking this icon will cause the user to be logged in to Windows without being prompted for a Windows user password.
  2. Wenger - Clicking this icon will cause the user to be logged in to Windows only after he supplies the password for the “Wenger” Windows user account.

When the user logs in to Windows using the “Operator” Windows user account, Windows will automatically start up the APM application software (which may require up to a minute to completely start up and begin running), and automatically login the user at the “Operator” APM software level.

When startup is complete the APM software's Main screen will appear (see Figure 2.2). The various features of this Main screen will be explained in detail in a later chapter, but for now notice that four of the buttons (1), (2), (3) and (4) in the lower region of the screen are disabled. That is because users at the “Operator” APM software level are restricted from accessing these features. These buttons, however, are all enabled for users at the “Wenger” APM software level.

Also, notice in Figure 2.3 below, that the current APM software user account, “Operator” in this case (1), is displayed in the upper left region of the Main screen.

2.1.5 Switching APM Software User Types

As already stated, the APM software recognizes the following four APM software user accounts in ascending order of security levels and privileges:

  • Operator
  • Maintenance
  • Supervisor
  • Wenger

As an example, the “Supervisor” APM software user account allows the user to modify the various PID control loop tuning values, something not permitted by the “Operator” or “Maintenance” APM software user accounts. When the customer's operators are logged in and using the APM software to control the extruder system, occasionally customer personnel with more expansive user privileges may want to use the HMI for a short time. The APM software allows for doing this by providing the “(csL) Log On” button in the lower right region on the Main screen (see Figure 2.4). When this button is clicked, the FactoryTalk View SE Client Login dialog box appears with which the user can switch to a APM software user level with more privileges, by specifying a different APM software user account in the “User name” text box and the associated password in the “Password” text box.

Each of the APM software user accounts other than the “Operator” user account has its own unique associated password. After the user clicks the “(csL) Log On” button, and then log on using one of the three APM software user accounts with elevated security level and completes what he wants to do, he can easily revert back to the “Operator” APM software user account by clicking the “Log Off” button.
